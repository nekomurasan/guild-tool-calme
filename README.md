# BrownDust Ⅱ Calme Guild Tool

ギルド運営に役立つツールをまとめたポータルです。現在は以下の2つを収録しています。

- **Ace Characters Database**: オーバーキルスコアを目指せるキャラクターの育成状況管理ツール
- **Gold Farming Guide**: 商店・料理を利用した金策ガイド

## ファイル構成

```
/index.html                 … ポータル(トップページ)。ツール一覧を表示
/style.css                  … ポータルページの見た目
/equipment/index.html       … 「Ace Characters Database」の画面構造
/equipment/style.css        … 「Ace Characters Database」の見た目
/equipment/app.js           … 「Ace Characters Database」の動作(Firebase連携など)
/gold-farming/index.html    … 「Gold Farming Guide」の画面構造
/gold-farming/style.css     … 「Gold Farming Guide」の見た目
/gold-farming/app.js        … 「Gold Farming Guide」の動作(Firebase連携など)
/gold-farming/icons/        … 食材アイコン画像(ファイル名 = 食材名)
```

GitHub Pagesにアップロードする際は、このフォルダ構成のまま(`equipment`フォルダ・`gold-farming`フォルダをそれぞれ作って中に3ファイル、直下にトップの`index.html`・`style.css`)アップロードしてください。

**今後ツールを追加する場合**は、`/damage-calc/` のように新しいフォルダを作り、トップの `index.html` にあるツール一覧(`.tools` の中)にカードを1つ追加するだけで済むようにしてあります。

## Ace Characters Database について

オーバーキルスコアを目指せるキャラクターについて、「誰が」「どのキャラを」「武器・鎧・頭・装飾・腕のどの部位でどの等級まで育成済みか」をギルドメンバー自身が登録・共有できるツールです。

- 1キャラクターにつき、武器・鎧・頭・装飾・腕の5部位を登録可能(1部位以上の入力が必須)
- 各部位は、汎用装備(30種類の固定リスト)または「専用装備」から選択
- 装備を選んだ部位は、等級(18〜24)の入力も必須
- 備考欄(20文字まで)で補足情報を残せる
- 編集・削除は「登録する」画面の一時リストでのみ行う運用(一覧画面はあくまで閲覧用)
- 特定のキャラ(現在は「グランヒルト」)には、関連する他のキャラ(ディアナ・ルゥ)の登録状況もまとめて見られる「関連キャラ」ボタンが付く

### members の中身の例

```json
{
  "name": "つき",
  "exclusive": [
    {
      "character": "シェラザード",
      "slots": {
        "武器": { "item": "専用装備", "grade": "23" },
        "鎧":   { "item": "海神の鱗", "grade": "18" },
        "頭":   { "item": "", "grade": "" },
        "装飾": { "item": "毒蛇の手", "grade": "24" },
        "腕":   { "item": "守護の龍鱗", "grade": "23" }
      },
      "note": "りゅうりん24予定11月"
    }
  ],
  "updatedAt": "2026-07-13T12:00:00.000Z"
}
```

`slots` は武器・鎧・頭・装飾・腕の5キーを持ち、各値は `{ item, grade }`。`item` が空文字の場合はその部位は未登録扱い。

### 関連キャラの設定

`equipment/app.js` 内の `RELATED_CHAR_GROUPS` という定数で管理しています。

```js
const RELATED_CHAR_GROUPS = {
  "グランヒルト": ["グランヒルト", "ディアナ", "ルゥ"]
};
```

キーに指定したキャラクターの行にのみ「関連キャラ」ボタンが表示され、押すと配列内の全キャラの登録状況(同じ所持者のもの)がポップアップで表示されます。グループを増やしたい場合は、このオブジェクトに項目を追加してください。

### 権限・ログインの仕組み

- ログインしないとツール自体が開けません(メール/パスワード方式、実際にはID方式)
- ログインアカウントとギルド名簿の名前を紐付けておくと、ログイン時に自動でその名前が操作者にセットされ、他人の名前を選べなくなります(なりすまし防止)
- 「管理者」として登録されたアカウントのみ、以下の操作が可能です(Firestoreのルールでも制限済み)
  - ログインアカウントの紐付け・管理者アカウントの追加削除
  - メンバー名簿の削除・表示名の変更
  - 全データのバックアップ(JSONダウンロード)

管理者は「名簿管理」タブの「管理者アカウント」から追加・削除できます(自分が管理者としてログインしている場合のみ表示されます)。**この管理者アカウントは、Gold Farming Guideの編集権限とも共通です。**

## Gold Farming Guide について

商店での購入推奨リスト・料理の優先度リスト・売却してはいけないNGリストをまとめた、金策ガイドページです。**メインギルド・サブギルドどちらのメンバーも閲覧できます**が、閲覧の入り口が2種類あります。

### アクセス方法(2経路)

| 対象 | 方法 | 権限 |
|---|---|---|
| メインギルド | Ace Characters Databaseと同じID/パスワードでログイン | 管理者アカウントなら編集も可能 |
| サブギルド | ギルド内で共有する共通パスワードを入力 | 閲覧のみ(編集は不可) |

サーバー負荷や流出リスクを考慮し、サブギルド向けには個別アカウントを発行せず、共通パスワード1つで運用しています。共通パスワードはFirestoreの `goldFarming/config` ドキュメント(`password`フィールド)に保存されており、管理者アカウントでログインすれば変更できません(現状、変更UIは未実装。変更する場合はFirebaseコンソールから直接書き換えてください)。

### リストの内容

- **商店購入推奨リスト**: カセット名ごとに、購入すべき食材をアイコン付きバッジでまとめて表示。「購入なし」と入力した食材は赤字で強調され、該当カセット名もグレー表示になります
- **料理推奨リスト**: 優先度(★〜★★★★★★の6段階)と料理名。並び替え可能
- **売却してはいけないNGリスト**: アイテム名と備考(改行対応)。並び替え可能

各リストのタイトル下には、管理者が編集できるサブ説明文があります。

### 食材アイコンについて

`gold-farming/icons/` フォルダに、**食材名と完全に一致するファイル名**(例: `小麦.png`)の画像を置くと、商店購入推奨リストに自動で表示されます。アイコンが存在しない食材は名前のみ表示されます。

### アクセス状況の記録(簡易)

サブギルドの共通パスワードが外部に流出していないか確認するため、アクセスのたびに「日時」「メインギルド経由/サブギルド経由」だけをFirestoreの `goldFarmingAccessLogs` に記録しています(個人情報は記録しません)。管理者でログインすると、ページ内に本日・直近7日間・日別のアクセス件数が表示されます。**サブギルド経由のアクセス数が普段と比べて急増していたら、パスワード流出を疑い、変更を検討してください。**

### goldFarming コレクションの中身の例

```json
// goldFarming/config
{ "password": "共通パスワード文字列" }

// goldFarming/content
{
  "shopList": {
    "description": "サブ説明文",
    "rows": [ { "cassette": "カセットA", "ingredient": "小麦" } ]
  },
  "cookingList": {
    "description": "サブ説明文",
    "rows": [ { "priority": "★★★", "dish": "○○スープ" } ]
  },
  "noSellList": {
    "description": "サブ説明文",
    "rows": [ { "item": "○○の欠片", "note": "備考\n複数行対応" } ]
  }
}
```

## 使っている外部サービス

- **Firebase Firestore**: データの保存場所(データベース)。両ツールで同じFirebaseプロジェクトを共有しています
- **Firebase Authentication**: メインギルド向けログイン機能(メール/パスワード方式。実際のメールではなく `ID@calmeguild.local` という形式のダミーアドレスを使用)

設定情報は各ツールの `app.js` 先頭付近、`firebaseConfig` に書かれています(両ツールで同一の内容です)。

## Firestoreのデータ構造(全体)

| コレクション | ドキュメントID | 内容 | 使用ツール |
|---|---|---|---|
| `members` | メンバー名 | 登録キャラクター一覧(`exclusive`配列) | Ace Characters Database |
| `roster` | `members` / `characters` | メンバー名簿・キャラ名簿 | Ace Characters Database |
| `logs` | 自動採番 | 操作ログ | Ace Characters Database |
| `userLinks` | ログインID | ログインアカウントとメンバー名の紐付け | Ace Characters Database |
| `admins` | ログインID | 管理者アカウント | 両ツール共通 |
| `goldFarming` | `config` / `content` | 共通パスワード・リスト内容 | Gold Farming Guide |
| `goldFarmingAccessLogs` | 自動採番 | アクセス日時・経路の記録 | Gold Farming Guide |

## セキュリティ(Firestoreルール)

- `admins` / `userLinks`: 管理者だけが書き込み可能
- `goldFarming`: 誰でも読み取り可能(サブギルドの未ログイン閲覧のため)、書き込みは管理者のみ
- `goldFarmingAccessLogs`: 誰でも書き込み(記録)可能、読み取りは管理者のみ
- それ以外(`members` / `roster` / `logs`): ログインしていれば誰でも読み書き可能

Firebaseコンソールの「Firestore Database」→「ルール」で確認・変更できます。

## 運用でやっておくと良いこと

- **バックアップ**: 「名簿管理」タブの「データバックアップ」(管理者のみ表示)から、定期的に全データをJSONでダウンロードしておくと、誤操作でデータが消えても復元の手がかりになります(自動復元機能ではなく、あくまで控えです)
- **編集ログの整理**: テスト時の記録が気になる場合は、Firebaseコンソールの「Firestore Database」→「データ」タブから `logs` や `goldFarmingAccessLogs` コレクションを直接削除できます
- **サブギルド共通パスワードの管理**: 流出が疑われる場合は、Firebaseコンソールから `goldFarming/config` の `password` フィールドを書き換え、サブギルドメンバーに新しいパスワードを再共有してください

## 今後の機能追加について

新しい機能を既存ツールに追加する際は、上記のFirestore構造を踏まえた上で、それぞれの `app.js` に新しいタブ・関数を追加していく形になります。データの持たせ方に迷ったら、既存の構造を参考にすると一貫性が保てます。新しい独立したツールを追加する場合は、新しいフォルダを作ってポータルの `index.html` にリンクを追加してください。
