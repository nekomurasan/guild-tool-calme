# BrownDust Ⅱ Calme Guild Tool

ギルド運営に役立つツールをまとめたポータルです。現在はキャラクター管理ツール「Ace Character Database」を収録しています。

## ファイル構成

```
/index.html              … ポータル(トップページ)。ツール一覧を表示
/style.css               … ポータルページの見た目
/equipment/index.html    … 「Ace Character Database」の画面構造
/equipment/style.css     … 「Ace Character Database」の見た目
/equipment/app.js        … 「Ace Character Database」の動作(Firebase連携など)
```

GitHub Pagesにアップロードする際は、このフォルダ構成のまま(`equipment`フォルダを作って中に3ファイル、直下にトップの`index.html`・`style.css`)アップロードしてください。

**今後ツールを追加する場合**は、`/damage-calc/` のように新しいフォルダを作り、トップの `index.html` にあるツール一覧(`.tools` の中)にカードを1つ追加するだけで済むようにしてあります。

## Ace Character Database について

オーバーキルスコアを目指せるキャラクターについて、「誰が」「どのキャラを」「武器・鎧・頭・装飾・腕のどの部位でどの等級まで育成済みか」をギルドメンバー自身が登録・共有できるツールです。

- 1キャラクターにつき、武器・鎧・頭・装飾・腕の5部位を登録可能(1部位以上の入力が必須)
- 各部位は、汎用装備(30種類の固定リスト)または「専用装備」から選択
- 装備を選んだ部位は、等級(18〜24)の入力も必須
- 備考欄(20文字まで)で補足情報を残せる
- 編集・削除は「登録する」画面の一時リストでのみ行う運用(一覧画面はあくまで閲覧用)
- 特定のキャラ(現在は「グランヒルト」)には、関連する他のキャラ(ディアナ・ルゥ)の登録状況もまとめて見られる「関連キャラ」ボタンが付く

## 使っている外部サービス

- **Firebase Firestore**: データの保存場所(データベース)
- **Firebase Authentication**: ログイン機能(メール/パスワード方式。実際のメールではなく `ID@calmeguild.local` という形式のダミーアドレスを使っています)

設定情報は `equipment/app.js` の先頭付近、`firebaseConfig` に書かれています。

## Firestoreのデータ構造

| コレクション | ドキュメントID | 内容 |
|---|---|---|
| `members` | メンバー名 | そのメンバーが登録したキャラクター一覧(`exclusive`配列) |
| `roster` | `members` | ギルドメンバーの名前一覧(`names`配列) |
| `roster` | `characters` | 属性ごとのキャラクター一覧と攻撃タイプ(💧水/🔥火/🍃風/🌟光/🟣闇 それぞれに `{name, atkType}` の配列) |
| `logs` | 自動採番 | 誰が・いつ・何をしたかの操作ログ |
| `userLinks` | ログインID(内部形式) | ログインアカウントとメンバー名の紐付け |
| `admins` | ログインID(内部形式) | 管理者として登録されているアカウント |

### members の中身の例

```json
{
  "name": "つき",
  "exclusive": [
    {
      "character": "シェラザード",
      "slots": {
        "武器": { "item": "専用装備", "grade": "23" },
        "鎧":   { "item": "海神の鱗", "grade": "18" },
        "頭":   { "item": "", "grade": "" },
        "装飾": { "item": "毒蛇の手", "grade": "24" },
        "腕":   { "item": "守護の龍鱗", "grade": "23" }
      },
      "note": "りゅうりん24予定11月"
    }
  ],
  "updatedAt": "2026-07-13T12:00:00.000Z"
}
```

`slots` は武器・鎧・頭・装飾・腕の5キーを持ち、各値は `{ item, grade }`。`item` が空文字の場合はその部位は未登録扱い。

### 関連キャラの設定

`equipment/app.js` 内の `RELATED_CHAR_GROUPS` という定数で管理しています。

```js
const RELATED_CHAR_GROUPS = {
  "グランヒルト": ["グランヒルト", "ディアナ", "ルゥ"]
};
```

キーに指定したキャラクターの行にのみ「関連キャラ」ボタンが表示され、押すと配列内の全キャラの登録状況(同じ所持者のもの)がポップアップで表示されます。グループを増やしたい場合は、このオブジェクトに項目を追加してください。

## 権限・ログインの仕組み

- ログインしないとツール自体が開けません(メール/パスワード方式、実際にはID方式)
- ログインアカウントとギルド名簿の名前を紐付けておくと、ログイン時に自動でその名前が操作者にセットされ、他人の名前を選べなくなります(なりすまし防止)
- 「管理者」として登録されたアカウントのみ、以下の操作が可能です(Firestoreのルールでも制限済み)
  - ログインアカウントの紐付け・管理者アカウントの追加削除
  - メンバー名簿の削除・表示名の変更
  - 全データのバックアップ(JSONダウンロード)

管理者は「名簿管理」タブの「管理者アカウント」から追加・削除できます(自分が管理者としてログインしている場合のみ表示されます)。

## セキュリティ(Firestoreルール)

`admins` / `userLinks` は管理者だけが書き込み可能、それ以外(`members` / `roster` / `logs`)はログインしていれば誰でも読み書き可能、という設定です。Firebaseコンソールの「Firestore Database」→「ルール」で確認・変更できます。

## 運用でやっておくと良いこと

- **バックアップ**: 「名簿管理」タブの「データバックアップ」(管理者のみ表示)から、定期的に全データをJSONでダウンロードしておくと、誤操作でデータが消えても復元の手がかりになります(自動復元機能ではなく、あくまで控えです)
- **編集ログの整理**: テスト時の記録が気になる場合は、Firebaseコンソールの「Firestore Database」→「データ」タブから `logs` コレクションを直接削除できます

## 今後の機能追加について

新しい機能(ダメージ計算、理想パーティ提案など)を既存のAce Character Databaseに追加する際は、上記のFirestore構造を踏まえた上で、`equipment/app.js` に新しいタブ・関数を追加していく形になります。データの持たせ方に迷ったら、既存の `members` / `roster` の形を参考にすると一貫性が保てます。新しい独立したツールを追加する場合は、新しいフォルダを作ってポータルの `index.html` にリンクを追加してください。
